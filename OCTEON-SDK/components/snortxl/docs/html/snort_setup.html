SnortXL Setup

1. Introduction

SnortXL in Simple Executive user mode can be run as a Network Intrusion Detection System (NIDS)for Network Traffic and in pcap file read mode to analyze pcap files. In NIDS mode, SnortXL acts as a gateway between LAN and WAN networks. The ports on OCTEON board are divided into LAN and WAN groups. Any traffic which comes from a port in LAN group is analyzed by SnortXL IDS and passed on to the corresponding port in WAN group and vice-versa.

In the illustration below, port 0 belongs to LAN group and Port 1 to the WAN group, thus constituting one pair of LAN-WAN ports. Traffic coming on these ports will be toggled between them.

     L              SnortXL running on OCTEON®                 W
     A             +--------------------------+                A
     N             |                          |                N
           port  0 |                          | port  1
     S    ---------+ <======================> +---------       S
     I             |                          |                I
     D             |                          |                D
     E             +--------------------------+                E

The same setup can be extended to other deployment scenarios - In cases where SnortXL is deployed in front of a web server, one set of ports can be connected to the server and the other set of ports to receive client requests.

This page describes the setup required to run SnortXL on OCTEON® CN70XX EVB for Network Traffic. The setups and diagrams mentioned in this documentation are for illustrative purposes only. The end users may choose to setup OCTEON® for SnortXL as per their requirements.

2. Pre-requisites

Before proceeding further it is assumed that

  • OCTEON-SNORTXL-x.y.z rpm is installed on the host machine.
  • SnortXL is compiled as per the steps mentioned in Building and Running SnortXL.

3. Setup Details

The setup for SnortXL using OCTEON® CN70XX EVB can be configured as below settings:

3.1 Setup Requirements

   1. OCTEON® EVB 70XX
   2. Serial console cable.

3.1.1 EVB Setup Requirements

Board Details 

    Board Details
    Processor     : OCTEON CN70XX EVB
    Core clock    : 1500 MHz
    IO clock      : 600 MHz
    DDR clock     : 667 MHz (1334 Mhz DDR)
    DRAM          : 2 GB

SnortXL requires a minimum of 2 GB of RAM. Linux should be provided with atleast 1024 MB of memory as part of the 'mem=' argument of the boot command. Of the remaining memory, 700 MB should remain unallocated to be available as bootmem for use by SnortXL/HFA.

4. Calculating Performance of SnortXL:

SnortXL reports throughput as Pkts/sec in PCAP mode. To calculate performance throughput in Mbps from Pkts/sec use the following formula:

Performance(Mbps) = [{(Pkts/sec) * AvgPktSize * 8} / 1000000]

For inline mode, refer to performance as reported by the traffic-generator.
